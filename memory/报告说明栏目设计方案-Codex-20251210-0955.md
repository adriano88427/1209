# 报告说明栏目设计方案

目标：在每份 HTML 报告尾部追加统一的“报告说明”板块，便于溯源、解释口径、明确入榜规则与评分方式，保证不同报告间可比。

## 模块覆盖
- 单因子分析（非参数）、带参数的单因子分析（10等分）、带参数的单因子自由区间挖掘。
- 双因子分析（非参数）、带参数的双因子自由区间挖掘。

## 栏目结构与内容
1) 运行与调试信息（表格呈现）
   - 分析结果数量（有效区间/因子数）
   - 日志文件路径（最新 factor_analysis_log_*.txt）
   - 调试模式：开启/未开启；调试原始日志：已生成/未生成
   - 分档策略与样本下限（示例：quantile，默认分档 8；样本下限 1051）
   - 过滤统计（总区间 X，样本不足 Y，其他跳过原因）
2) 口径与公式（列表）
   - 年化收益率：日均收益 × 252（线性年化，不做复利）
   - 夏普比率：日均收益 ÷ 日标准差 × √252（未计算则标明）
   - 最大回撤：基于日收益序列的峰谷最大回撤
   - 平均每笔收益率：单笔收益均值
   - IC 口径：默认 Spearman（Rank IC）；如切换 Pearson 需显式标注
3) 榜单/入榜规则（列表）
   - 样本下限（示例：单因子自由区间 auto=总样本/20；双因子参数化 500；若有协同增益阈值>20% 也注明）
   - 其他过滤：年化收益率阈值（如整体×系数），用户自定义区间是否强制展示
   - 展示条数：Top N（若倍增展示需说明）
4) 评分与排名方式（列表）
   - 评分项与权重（收益/夏普/回撤/样本量等），样本量得分的加减规则或分位插值方式
   - 排序字段：按综合得分降序；若有相邻平滑得分，说明用途
   - 高亮规则：指标前 3 蓝色；用户区间黄色；告警/异常标注说明

## 数据来源与填充逻辑
- 元数据：运行时动态获取（日志路径、结果数量、分档/样本下限、过滤统计、是否调试）。
- 口径/公式：从现有配置和代码常量读取，保持与实际计算一致；若模块未计算某指标，标注“未启用”。
- 入榜规则/阈值：使用当前模块配置（样本下限、收益/协同增益阈值、用户区间强制展示）。
- 评分与权重：读取模块内的权重配置和得分算法描述，避免手写导致偏差。

## 展示形式
- “报告说明”单独成节，放在报告末尾。
- 运行与调试信息用 2 列表格；其余用短列表分条说明。
- 文案用简短中文，避免术语堆砌；遇到专业概念在行尾括注简释。

## 实施要点
- 所有 HTML 报告统一添加该节，模板可共用一段渲染函数。
- 未启用的指标/开关需显式写“未开启/未生成”，避免空白。
- 与《项目功能与配置使用说明.md》中的口径保持一致（特别是年化收益率线性口径）。
