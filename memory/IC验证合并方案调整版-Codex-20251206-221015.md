# IC验证合并方案（调整版，风险约束版）

## 调整原则
- 默认不改主路径：不开启会改变分布的开关，等权 IC 保持基线。
- 开关少而精：仅暴露必要验证开关（外部基准、CI/Bootstrap、零价审计、审计摘要落盘），默认关闭，核心因子优先。
- 输出轻量：单一审计摘要文件，差异/对比表有限几份，避免多表膨胀。

## 调整后的功能范围
1) 统一入口（fa_ic_validation.run_ic_validation）
   - 输入：基线汇总、对照汇总、审计日志；可选核心因子列表。
   - 输出：差异表（等权/加权/样本/skip）、核心因子对比表、审计摘要。
2) 审计摘要（默认开启）
   - 去重/跳过/缩尾/flip/零价过滤计数写入单一 `baogao/audit_summary_latest.csv`。
   - 零/负价处理：默认“仅计数+剔除”，不改值，需显式开关才清洗。
3) 外部基准对照（默认关闭）
   - 仅对核心因子对比，产出 `baogao/ic_benchmark_compare.csv`；适配层可插拔。
4) 核心因子 CI/Bootstrap（默认关闭）
   - 仅核心因子计算 CI/Bootstrap，输出 `baogao/ic_ci_core_factors.csv`；避免全量耗时。
5) 覆盖提示
   - 在汇总/报告增加年度覆盖、板块集中提示（轻量文本/字段，不新增大表）。

## 通过标准与回退
- 核心因子 |ΔIC| < 0.01 且方向一致；零/负价过滤计数可接受；无显著性能回归。
- 回退：关闭外部基准、CI/Bootstrap、零价清洗，保留差异表与审计摘要；恢复基线配置（无缩尾、可拼窗）。
